## What is Marimo?
Marimo is basically Jupyter notebooks but with a glow-up it's a reactive Python notebook that saves as regular `.py` files instead of messy JSON, so Git actually works without giving me merge conflict headaches. The whole vibe is reactivity: change a slider or tweak some code in one cell, and every cell that depends on it updates automatically. No more manually re-running everything or wondering if your outputs are stale. It comes with built-in UI widgets like sliders and dropdowns, so me can build little interactive tools without touching web dev. Since notebooks are pure Python files, me get full IDE support, clean diffs, and zero hidden state, what me see is exactly what's there. Perfect for data analysis that needs to be reproducible, shareable, and actually make sense when me come back to it next week.

## Git?

## What is Plotly?
I learned that Plotly is a visualization library that turns data into interactive, publication-quality charts that actually do things. I can hover to see values, zoom in on details, click to filter, or download as an image. It comes in two flavors: Plotly Express for when I want beautiful plots with minimal code (literally one line to get a scatter plot), and Graph Objects when I need complete control over every detail. The library handles everything from basic line charts for trends and bar charts for comparisons to scatter plots for correlations, histograms for distributions, and even 3D plots or maps. I can fully customize charts with colors, themes, labels, and annotations, and they play perfectly with Polars DataFrames. I can wrangle my data in Polars then pipe it straight into Plotly. I can export as interactive HTML files that anyone can open in a browser, or as static images for papers. The whole vibe is exploratory analysis meets polished presentation without the usual trade-off between quick and professional.

## What is polars?
I learned that Polars🐻‍❄️ is a DataFrame library built for speed and efficiency, it's written in Rust, runs operations in parallel by default, and is about 5-10x faster than Pandas while using less memory. The whole vibe is "fast data wrangling without the frustration." I can read data from CSV, JSON, or Parquet files, then start exploring with methods like `head()`, `describe()`, and checking the schema. When I need to select columns, I use `select()` with `pl.col()` expressions, and for filtering rows I chain conditions with `&` and `|`. Adding new columns happens with `with_columns()`, where I can do math operations, string manipulations, or even conditional logic using `when().then().otherwise()`. Grouping and aggregating with `group_by().agg()` lets me calculate means, counts, or multiple statistics at once. If I need to combine datasets, joins work exactly how I'd expect with `left`, `inner`, or `outer` options. For missing values, I can detect them with `null_count()`, drop them with `drop_nulls()`, or fill them using strategies like mean or forward fill. The real power move is method chaining—I can read, filter, transform, group, and sort in one readable pipeline. And when I'm working with massive datasets, I switch to lazy mode with `scan_csv()` and only call `.collect()` at the end, letting Polars optimize the whole query behind the scenes.

## what is a uv package?
I learned that UV is a Python package manager written in Rust that's basically pip on steroids...it's 10-100x faster, handles virtual environments automatically, and replaces a whole toolbox of separate tools. The whole vibe is "one tool to rule them all": it manages Python versions, creates virtual environments, installs packages, and runs my code without me ever needing to activate or deactivate anything. When I need a package, I just run `uv add polars` and it automatically installs it and updates my config file. To run my code, I use `uv run python script.py` and it just works, no more forgetting to activate environments or wondering which Python I'm using. The magic is that everything is reproducible: the `uv.lock` file locks exact versions so anyone cloning my project can run `uv sync` and have exactly the same setup.

# What is data science?
## what am I learning here? 😆😆
Data science is basically turning raw data into real insights that help people make better decisions, it's a mix of statistics, programming, and actually understanding the problem you're trying to solve. The workflow usually goes: collect messy data, clean it up, explore it to find patterns, build models that can predict or classify things, then explain what you found to people who don't code. What makes it useful is that organizations can stop guessing and start knowing—retailers predict what to stock, hospitals identify at-risk patients, banks catch fraud, and cities optimize traffic. On the technical side, I need Python for manipulation, stats for understanding uncertainty, visualization for showing patterns, and machine learning for predictions. But the soft skills matter just as much—asking the right questions, communicating clearly, and understanding the domain I'm working in. The field touches everything from Netflix recommendations to climate modeling to drug discovery, which is honestly pretty cool.

## What is markdown writing, why is it important?
Markdown is basically a cheat code for formatting text without ever touching a mouse instead of clicking buttons, I just wrap words with symbols like ** for bold, * for italics, and # for headings, and it magically looks clean and professional. It's used everywhere: GitHub, Reddit, Slack, Jupyter notebooks, basically any place where people write and share text. Headings go from one to six # symbols (one is biggest, six is smallest), lists can be bullet points with - or numbered with 1. 2. 3., and I can even make checklists with [x] and [ ]. For links it's text, for images it's the same but with an ! at the start, and for code I use backticks for inline stuff or triple backticks for blocks with language names like python for syntax highlighting. Tables are made with | pipes and - dashes, blockquotes with >, and horizontal lines with ---. The whole vibe is "write once, look good everywhere" and the files are just plain text so they'll open forever. Once I get used to typing bold instead of reaching for the toolbar, I dont think I'll ever go back😂.
